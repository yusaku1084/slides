# slides リポジトリ

## 概要

社内LTや勉強会のスライドをMarpで管理するリポジトリ。
Claude Codeと一緒に構成から考えてスライドを作成する。

---

## リポジトリ構成

```
slides/
├── CLAUDE.md                    ← このファイル
├── README.md
├── .claude/
│   └── skills/
│       └── marp/
│           └── marp.md          ← Marpスライド生成スキル
├── .github/
│   └── workflows/
│       └── deploy.yml           ← 自動ビルド & GitHub Pages公開
└── talks/
    └── YYYY-MM-テーマ名/
        ├── slides.md            ← Marpスライド本体
        └── assets/              ← 画像など
```

---

## スライド作成フロー

1. 発表テーマ・対象・ゴールをユーザーと一緒に整理する
2. 構成（目次）を考えてユーザーに確認する
3. 各スライドの内容を詰める
4. `talks/YYYY-MM-テーマ名/slides.md` を生成する
5. `talks/YYYY-MM-テーマ名/assets/` に必要な画像を配置する

---

## フォルダ命名規則

```
talks/YYYY-MM-テーマ名/
例：
talks/2026-04-jujutsu/
talks/2026-05-react-server-components/
talks/2026-06-claude-code/
```

---

## スライド作成のルール

- スライド生成前に必ず `.claude/skills/marp/marp.md` を読むこと
- スタイルは `.claude/skills/marp/marp.md` に定義されたものを使う
- 1スライド1メッセージを意識する
- コードブロックは必ず言語を指定する
- 図はASCIIアートかMermaidで表現する（画像は最小限）

---

## やってほしいこと

- 構成を提案するときは目次形式で見せる
- ユーザーの確認を取ってから各スライドの内容を詰める
- スライドの話す内容（トークスクリプト）もセットで提案する
- 時間配分も意識する（5分・8分・20分など）

## やってほしくないこと

- 確認なしにいきなりスライドを生成しない
- スタイルを勝手に変更しない
- 1スライドに情報を詰め込みすぎない
- 背景色と文字色のコントラストを必ず確認する（グラデーション背景のスライドは特に注意）
- コードブロック（pre）がある場合、背景色付きセクションでの視認性を確認する
